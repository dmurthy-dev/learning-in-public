## Issues encountered (blocking)

### 1. File handling limitation (Google Drive)
Execution failed with the following error:
```
System.InvalidOperationException:
Cannot download folders. Only files currently supported.
```

**Cause:**
- The **Get File or Folder** activity was pointing to a Google Drive **folder**, not a file.
- UiPath’s Google Drive connector does not support folder downloads for downstream activities.

**Workaround applied:**
- Imported Microsoft-format files (Excel) and left them unchanged instead of converting to Google Sheets.
- This allowed partial progress but introduced downstream incompatibilities.

---

### 2. Lab environment mismatch (core constraint)

This lab assumes OneDrive and Excel Online connectors.
My setup uses Google Drive, Google Sheets, and Gmail.

**Key consequences:**
- Excel Online activities cannot be directly replaced with Google Sheets equivalents.
- Semantic Similarity and file grounding depend on Excel Online row iteration.
- File grounding in GenAI activities expects supported file types, resulting in:
```Unsupported file type: unknown (422 Unprocessable Entity)```

## Key learnings

### 1. Template ≠ drop-in workflow
Prebuilt templates often:
- Assume a specific cloud ecosystem.
- Require customization to function elsewhere.
- Are best treated as **reference implementations**, not copy-paste solutions.

---

### 2. Context grounding (conceptual clarity)
Context grounding means providing the AI with:
- Relevant background data
- Structured inputs (files, rows, or prior outputs)

Without grounding, AI outputs are generic.  
With grounding, outputs are more accurate and consistent.

In this lab:
- Context grounding was designed around Excel Online rows.
- This does not translate cleanly to Google Sheets in Studio Web.

---

### 3. Block structure affects downstream behavior
- Activities inside conditions execute only when conditions are met.
- Activities outside blocks may run with missing or null data.
- Disabled or partially configured steps can silently impact later stages.

---

### 4. Integration boundaries are the weakest links
Even when prompts and logic are correct, workflows can fail due to:
- Connector limitations
- Unsupported file types
- Platform-specific assumptions

Automation failures often occur at **integration boundaries**, not core logic.
